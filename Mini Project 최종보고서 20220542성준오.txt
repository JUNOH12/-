선박 부품 재고 관리 시스템 설계

 프로젝트 배경
선박 운항 및 유지보수 과정에서는 다양한 종류의 부품이 지속적으로 사용된다.
이러한 부품들은 관리 주기가 길고 종류가 많아, 수기 관리나 단순한 엑셀 기반 관리 방식으로는 재고 누락, 과잉 발주, 부품 부족 등의 문제가 발생할 수 있다.
이에 본 프로젝트에서는 선박 부품을 체계적으로 관리할 수 있는 재고 관리 시스템을 설계하여, 부품의 등록부터 재고 현황 확인, 발주 관리까지 효율적으로 수행할 수 있도록 하고자 한다.

Mini Project 산출물

본 Mini Project에서는 다음의 네 가지 산출물을 중심으로 시스템을 설계하였다.

산출물 ① 업무정의서

본 시스템은 선박 부품의 등록, 재고 관리, 발주 업무를 통합적으로 관리하기 위한 시스템이다.
사용자는 부품 정보를 등록하고 재고 수량을 확인할 수 있으며, 관리자는 재고 부족 시 발주 요청을 통해 부품을 주문할 수 있다.

업무 흐름은 다음과 같다.
사용자가 부품을 등록하고 재고 정보를 관리하며, 재고 수량이 기준치 이하로 감소할 경우 발주 요청을 수행한다. 시스템은 해당 이력을 기록하여 추후 관리가 가능하도록 한다.

산출물 ② Use Case Diagram

Use Case Diagram은 시스템 사용자와 기능 간의 관계를 표현하기 위해 작성하였다.

Actor: 관리자, 사용자

주요 Use Case:

부품 등록

부품 조회

재고 관리

발주 요청

발주 처리

본 다이어그램을 통해 시스템의 기능 범위와 사용자 역할을 명확히 정의하였다.

산출물 ③ Class Diagram 및 Sequence Diagram

Class Diagram에서는 시스템을 구성하는 주요 클래스와 그 관계를 설계하였다.
주요 클래스는 부품, 재고, 사용자, 발주, 카테고리로 구성되며, 각 클래스는 명확한 책임을 갖도록 설계하였다.

Sequence Diagram은 재고 부족 상황 발생 시 발주 요청이 이루어지는 흐름을 중심으로 작성하였다.
사용자, 시스템, 발주 객체 간의 메시지 흐름을 시간 순서대로 표현하여 시스템 동작 과정을 직관적으로 이해할 수 있도록 하였다.

산출물 ④ ER Diagram 및 UI 설계

ER Diagram을 통해 데이터베이스 관점에서의 구조를 설계하였다.
부품, 재고, 사용자, 발주, 위치 엔티티를 정의하고 각 엔티티 간의 관계를 설정하여 데이터 중복을 최소화하였다.

UI 설계는 실제 사용 환경을 고려하여 단순하고 직관적인 구조로 구성하였다.
대시보드, 부품 관리 화면, 재고 관리 화면, 발주 관리 화면으로 구성하여 사용자가 시스템을 쉽게 이해하고 사용할 수 있도록 설계하였다.

결론 및 느낀 점

본 Mini Project를 통해 시스템을 설계할 때 요구사항 분석과 구조적 설계의 중요성을 이해할 수 있었다.
Visual Paradigm을 활용하여 UML 다이어그램을 작성함으로써 실제 현업에서 사용되는 설계 방식을 경험할 수 있었으며, 향후에 해사 IT 및 해사 분야에서 시스템 설계 업무를 수행하는 데 기초적인 역량을 기를수있었다.
